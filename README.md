## 🚀 OverviewThis project implements a zero-touch, closed-loop Security Orchestration, Automation, and Response (SOAR) pipeline. It bridges the gap between high-fidelity threat detection and infrastructure-level containment, reducing the Mean Time to Respond (MTTR) to near-zero.  ## 🏗️ System ArchitectureThe architecture follows a decoupled, microservices approach to defense-in-depth:  Detection Layer: Suricata (Network IDS) and Wazuh (Host IDS) provide raw telemetry.  Analytical Brain: ANAOS SIEM Engine ingests logs, applies stateful thresholding, and enriches data with MITRE ATT&CK tags (T1595, T1110).  Orchestration: Shuffle SOAR polls the ANAOS API and applies boolean logic to filter noise.  Enforcement: Ansible dynamically alters the pfSense firewall's kernel routing tables to block attackers at Layer 3.  [Place ARCH.jpg or ARCH2.jpg here]## 🛠️ Technical Deep Dive1. Analytical Thresholding (ANAOS API)Rather than hardcoding responses, ANAOS exposes stateful alerts via a structured JSON REST API (/api/alerts).  Logic: Ingests flat logs and applies sliding window thresholding (e.g., X failed logins in Y seconds).  Output: Generates a JSON payload containing the severity level and the critical primary key: the attacker's IPv4 address.
